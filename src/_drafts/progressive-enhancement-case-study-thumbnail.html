You've come across them countless times on blogs, social media and especially webshop pages: small images with zooming, image gallery navigation, descriptions, captions, all kinds of features. They work so well we barely take notice, except when they don't. Like when they are slow as hell, cause hiccups or completely break the webpage, forcing us to reload.

In this article I explore progressive enhancement applied to user experience with image thumbnails. I start from the bare minimal concept and walk briefly through the additional layers on top, to arrive at the progressively enhanced version most of us take for granted today. Each layer enables extra or enhances existing affordances, which in turn improve user experience. Try it for yourself by clicking the thumbnails in this article and comparing the level of comfort/ irritation you experience!

Let's get it over with the definitions already and make sure we're all on the same page:

progressive enhancement
Progressive enhancement is a design philosophy that centers around providing a baseline of essential content and functionality to as many users as possible, while at the same time going further and delivering the best possible experience only to users of the most modern browsers that can run all the required code. (via developer.mozilla.org)
thumbnail
a small image that represents a larger version of the same image, and also usually serves as a link to its larger counterpart (adapted from techterms.com)
affordance
the quality or property of an object that defines its possible uses or makes clear how it can or should be used (via merriam-webster.com)

So what are the bare affordances of a thumbnail?

  1. Cognitive: it should be apparent that, and how, a thumbnail will react to user actions, for both desktop and mobile users (mouse, keyboard and touch).
  2. Functional: The action of clicking or tapping the thumbnail should display the larger image.
  3. Functional: The action of tabbing to the thumbnail on desktop and pressing Enter should display the larger image.

Cross-browser link (<a>) behavior already satisfies the functional requirements, but the thumbnail you see below has no indication that it can be expanded to full size upon clicking it. Desktop users do see the cursor changing to a pointer (hand) but that's still a pretty vague indication.

The functional thumbnail does not provide the basic cognitive affordances of a good thumbnail.

A basic thumbnail

Thumbnail

Code

<a href="path/to/thumbnail.png" title="View full size" target="_blank">
  <img src="path/to/full-size-image.png" alt="An image">
</a>

In this version I've added a title attribute which says View full size. Better: when putting your cursor over the image on desktop, you know what to expect when clicking it. It's still not apparent on first view that the image is clickable though, especially not on mobile. I will enhance this in the next version.

There's also (in the context of this article) a minor annoyance in that clicking the 'functional' thumbnail will redirect you away from this article in the same tab, forcing the user to reload the current page if (s)he wishes to come back. We need to provide a new affordance: interacting with the thumbnail should not, or as little as possible, interrupt the reader flow. Thanks to the target="_blank" attribute, the full-size image will now open in a new tab.*

* Whether opening a link in a new tab by default is good or bad is admittedly debatable, and depends on a lot of factors including type of website, type of content, device, OS, target audience. But I found opening in a new tab much more user-friendly on mobile (when you tap 'Go back' on Android, it will close an image tab).

A thumbnail with action indicator

In our mobile-first world, there must be a way for mobile users to identify that they can expand the thumbnail to full size by clicking it. I've added 2 versions here: I could use simple text (safe but plain), or an icon (more concise but the meaning of the pictogram should be almost web-universally agreed upon) or both (adds visual clutter).

This is about the most advanced functionality you can get without using Javascript (or even CSS), and also provides all basic thumbnail affordances. The only area where user experience can be enhanced is not interrupting reader flow, but at what cost?

A Javascript-enhanced thumbnail

The thumbnail below is much more like what we know! It took about 40 lines of styling and 30 lines of Javascript code. It's much more user-friendly in the sense that it requires only: 1 click to expand the image, and 1 click to go back to the article without reloading the page. I've added a close icon in the top-right, and keyboard users can even hit ESC to close the overlay.

Additional optional UX experience enhancers would be image transitions, a loading spinner, or even widgets built on top like image galleries, with captions and more. While this is outside the scope of this article, you almost definitely encountered these from the FancyBox or LightBox libraries.

This article explored how progressive enhancement can be applied to thumbnail images, and how low-budget projects can provide a fully functional thumbnail feature without spending too many ressources. It also listed all affordances a thumbnail should provide:

  1. Cognitive: it should be apparent that, and how, a thumbnail will react to user actions, for both desktop and mobile users (mouse, keyboard and touch).
  2. Functional: The action of clicking or tapping the thumbnail should display the larger image.
  3. Functional: The action of tabbing to the thumbnail on desktop and pressing Enter should display the larger image.
  4. Additional: interacting with the thumbnail should interrupt the user flow as little as possible.
